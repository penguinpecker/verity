Request claims with the SDK; your user logs into myAadhaar inside a private browser Verity hosts — no extension, no install — and you receive attestor-signed answers with an on-chain record. Credentials and OTP never reach you.

    frontend — request claims
    

    import { requestIdentity } from '@verity/sdk/browser'

const res = await requestIdentity({
  flow: 'aadhaar-age',
  claims: ['age', 'name'],
})

res.pass          // true  — age gate cleared
res.claims        // { age_over_18, name, … } attestor-signed
res.proofs[0].tx  // recorded on Horizen mainnet

    01
    Hosted browser, no install
    Your user logs into myAadhaar inside a private, isolated browser Verity hosts — streamed into this page. No extension to install, no app to download.
    
      
    02
    Witness, don't collect
    The attestor witnesses the profile response over TLS with the session cookie redacted. Age is proven as a predicate — the DOB stays hidden unless the user approves disclosure.
    
      
    03
    Signed & recorded
    Each claim carries an attestor signature your backend can re-verify, and the age gate is recorded by a real transaction on the Horizen-mainnet verifier.
